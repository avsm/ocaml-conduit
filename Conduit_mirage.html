Module Conduit_mirage

module Conduit_mirage: sig .. end
Functorial connection establishment interface that is compatible with the Mirage libraries.

Currently supports two transports:

  • TCPv4 for remote communications using the TCPv4 protocol
  • Vchan for inter-VM communication within a single Xen host

server ~domid ~port ?read_size ?write_size will listen on a connection for a source domid and port combination, block until a client connects, and then return a t handle to read and write on the resulting connection. The size of the shared memory buffer can be controlled by setting read_size or write_size in bytes.

client ~domid ~port will connect to a remote domid and port combination, where a server should already be listening after making a call to Conduit_mirage.server. The call will block until a connection is established, after which it will return a t handle that can be used to read or write on the shared memory connection.

Close a Vchan. This deallocates the Vchan and attempts to free its resources. The other side is notified of the close, but can still read any data pending prior to the close.
